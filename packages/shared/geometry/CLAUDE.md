# geometry — 전개도↔UV 매핑·좌표 변환 (Model)

> 제품 요구사항(기능 ID 정의)은 루트 [CLAUDE.md](../../../CLAUDE.md)의 **PRD** 섹션 참조.

candle에서 가장 핵심적이고 까다로운 순수 함수 모음. 전개도와 3D를 잇는 수학.

## 담는 것
- shape별 **전개도 정의**: 원형/사각/하트 각각의 펼친 도면 레이아웃
- **UV 매핑**: 전개도의 한 점이 3D 메시 표면 어디에 붙는지 (실린더·박스·하트)
- **좌표 변환**: 화면(픽셀)↔전개도 좌표, 캔버스 pan/zoom 반영
- **회전 역변환**: 회전된 요소의 로컬 좌표계 변환 (대각선 스케일에 사용. 3D 직접배치 PRD-S2는 Could로 이관 — 착수 시 재사용)
- 규격(PRD-S5) 변경 시 전개도·UV 재계산

## 규칙
- 순수 함수만. UI·상태·부수효과 금지.
- 2D 에디터(`editor2d`)와 3D(`viewer3d`)가 같은 함수를 공유 — 좌표 로직을 중복 구현하지 말 것.

## 리스크
- **하트 등 비원통형 UV 매핑**이 난제. 신규 shape 추가 전 PoC로 검증.
