# share — 링크 진입·공유 (PRD-M5, PRD-S6)

> 제품 요구사항(기능 ID 정의)은 루트 [CLAUDE.md](../../../../CLAUDE.md)의 **PRD** 섹션 참조.

저장·링크 공유의 프론트 흐름. 편집은 소유 디자인(`/d/:id`, 로그인), 열람은 비로그인.

## 담는 것
- 소유 디자인 진입(`/d/:id`): 작성자가 원본을 다시 열어 수정(소유권 — `auth` 세션)
- 열람 링크 진입(`/view/:token`): 비로그인 사용자가 3D 시안 열람, **복제 후 수정**(로그인 필요)
- 저장 후 `/d/:id` 승격 + 발급된 열람 링크 노출
- 라우트 파싱(`/d/:id`·`/view/:token`·`/mypage`)

## 규칙
- 실제 토큰 발급·검증·소유권 판단은 백엔드(`apps/api/src/share`·`auth`·`designs`). 여기는 진입·UI 흐름.
- 저장·수정 버튼 배치는 App 셸, 로그인 세션은 `auth/`. 여기는 공유 링크 표현·복제 흐름.
- 열람자는 원본을 못 바꾸고, 복제본을 새 디자인(복제자 소유)으로 만들어 편집.
